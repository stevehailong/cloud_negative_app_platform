## 一、项目管理

### 1.1 项目与应用

**访问**: 导航栏 → 项目管理 / 应用管理

- 项目是应用的组织单元，一个项目下可以有多个应用
- 应用绑定代码仓库（GitLab），关联流水线、环境、发布

### 1.2 租户与组织

**访问**: 导航栏 → 组织架构（展开）

- 支持多租户、多组织层级

---

## 二、CI/CD 流水线

### 2.1 创建流水线

**访问**: 导航栏 → 流水线管理

1. 点击「新建流水线」
2. 选择应用，填写流水线编码（如 `common-ci-pipelines`）
3. 配置 GitLab 仓库地址、默认分支
4. 选择流水线类型：CI（仅构建）/ CD（仅部署）/ Full（构建+部署）

### 2.2 触发构建

- 点击「运行」手动触发
- 或配置 GitLab Webhook 自动触发
- Jenkins 自动执行：拉代码 → 编译 → 构建镜像 → 推送 Registry → 生成制品

### 2.3 查看制品

- 流水线执行成功后，制品表中出现对应镜像地址
- 镜像保存在 `172.18.0.1:5001/mycloud/<pipeline_code>:<tag>`
- Mac 终端拉取: `docker pull localhost:5001/mycloud/<image>:<tag>`

---

## 三、环境管理

**访问**: 导航栏 → 环境与基础设施 → 环境管理

- 支持 dev / test / staging / prod 四种环境类型
- 环境绑定集群和命名空间
- 应用环境绑定：关联应用 → 环境 → 模板

---

## 四、发布与部署

### 4.1 发布工单

**访问**: 导航栏 → 发布管理

1. 创建发布单：选择应用、目标环境、镜像版本、发布策略（滚动/蓝绿/金丝雀）
2. 提交审批 → 审批通过 → 执行发布
3. 支持回滚到历史版本

### 4.2 部署监控

**访问**: 导航栏 → 应用部署

- 查看 K8s Deployment 状态、Pod 列表、副本数
- 支持扩缩容、重启、回滚操作

---

## 五、集群管理

**访问**: 导航栏 → 环境与基础设施 → 集群管理

### 5.1 集群列表

- 显示纳管的 K8s 集群：名称、类型、版本、API Server、状态
- **同步节点**: 点击「集群详情」→ 自动从 K8s 同步节点信息

### 5.2 节点管理

- 点击「节点」按钮查看集群节点：名称、IP、角色、CPU、内存、OS、容器运行时
- 数据从 K8s API 实时同步

### 5.3 命名空间

- 点击「命名空间」查看集群命名空间
- 应用命名空间自动关联项目 ID 和应用名称
- 示例: `app-1-env-dev` → 项目ID=2, 应用="图书管理"

### 5.4 集群监控

- 点击「监控」查看集群概览：节点数、Pod 数、CPU/内存总量
- 节点详情表含实时指标

---

## 六、资源配额

**访问**: 导航栏 → 环境与基础设施 → 资源配额

- 支持多级配额：租户 → 项目 → 环境 → 应用 → 命名空间
- 限制项：CPU、内存、存储、Pod 数、Service 数、LB 数、GPU
- 示例: `app=图书管理` → CPU:4核, 内存:8Gi, Pod上限:10

---

## 七、配置管理

**访问**: 导航栏 → 系统管理 → 应用配置

- 应用级配置键值对管理
- 支持类型：string / number / boolean / json / yaml
- 版本化管理，按应用+环境隔离
- 示例: `图书管理/dev` → DB_HOST=172.18.0.1, DB_PORT=3306, GIN_MODE=release

---

## 八、密钥管理

**访问**: 导航栏 → 系统管理 → 应用密钥

- 密钥元数据管理（不存储明文）
- Vault 路径引用模式
- 按应用+环境隔离
- 示例: `图书管理/dev` → DB_PASSWORD → secret/book-service/dev/db-password

---

## 九、成本治理

**访问**: 导航栏 → 系统管理 → 成本治理

- 自动从 Prometheus 同步各命名空间的资源消耗
- 成本卡片：总成本、CPU 成本、内存成本、存储成本
- 支持按项目、应用、集群、日期筛选
- 示例: `app-1-env-dev` 日成本约 $0.004

---

## 十、监控中心

**访问**: 导航栏 → 监控中心

### 10.1 指标监控

- CPU 使用率、内存使用量、QPS、错误率
- 数据来源：Prometheus（cAdvisor / kubelet）
- 支持按应用筛选（自动关联 namespace）
- 时间范围：1h / 6h / 24h / 7d

### 10.2 日志查询

- 应用日志 / Pod 日志
- 支持关键词搜索、级别筛选

### 10.3 链路追踪

- 完整的请求链路可视化
- 支持按服务名筛选（如 "图书管理"）
- 水瀑布图展示各 Span 耗时
- 时间格式: `2026年06月05日 12时30分00秒`

---

## 十一、审计日志

**访问**: 导航栏 → 系统管理 → 审计日志

- 自动采集所有 API 请求（登录除外）
- 记录内容：用户、操作类型、资源、路径、IP、耗时、响应码
- 支持按用户、资源、操作筛选
- 支持导出

---

## 十二、链路追踪接入

### 12.1 自动采集

- Nginx access log → trace_json 格式 → trace-forwarder → monitor-service → MySQL
- 服务名自动映射：`book.com` → `图书管理`

### 12.2 查看 Trace

1. 访问「监控中心」→「链路追踪」
2. 选择服务名：`图书管理`
3. 点击「详情」查看水瀑布图

## 十三、K8s 运维命令参考

### 查看集群状态
```bash
kubectl get nodes
kubectl get pods -n app-1-env-dev
kubectl get deployments -n app-1-env-dev
```

### 同步集群数据到平台
```bash
# 同步节点
curl -X POST http://localhost:8088/api/v1/clusters/1/sync-nodes
# 同步命名空间
curl -X POST http://localhost:8088/api/v1/clusters/1/sync-namespaces
# 健康检查
curl http://localhost:8088/api/v1/clusters/1/health
```

### 拉取本地镜像
```bash
docker pull localhost:5001/mycloud/common-ci-pipelines:1.0.3578-a1b2c3d
```

### 访问 book.com 服务
```bash
curl http://book.com/api/books
curl -X POST http://book.com/api/books -H "Content-Type: application/json" -d '{"title":"测试","author":"作者","price":29.9}'
```

## 十四、故障排查

| 问题 | 排查命令 |
|------|----------|
| 服务不启动 | `docker compose ps` |
| 镜像拉取失败 | 检查 containerd certs.d 配置 |
| K8s 指标为0 | `curl localhost:8001` 确认 kubectl proxy 运行 |
| Trace 搜索为空 | 确认 serviceName 匹配（如 "图书管理"） |
| 权限不足 | 检查 iam_db.role_permissions 表 |
| 前端无数据 | 硬刷新 Cmd+Shift+R |
