# 发布管理环境选择功能实现

## ✅ 已实现功能

### 1. 动态加载已绑定环境
- 选择应用后，自动加载该应用已绑定的环境列表
- 环境选项显示：环境名称、环境类型、命名空间、集群名称
- 如果应用未绑定任何环境，显示提示信息

### 2. 环境必选校验
- 环境字段设置为必填项
- 未选择环境时无法提交发布
- 表单验证规则：`{ required: true, message: '请选择环境', trigger: 'change' }`

### 3. 只能选择已绑定的环境
- 环境下拉框只显示应用已绑定的环境
- 确保发布时使用正确的命名空间和集群

## 📋 使用流程

### 步骤1：创建发布
1. 进入"发布管理"页面
2. 点击【创建发布】按钮

### 步骤2：选择应用
1. 在"应用"下拉框中选择要发布的应用
2. 系统自动加载该应用已绑定的环境

### 步骤3：选择环境
1. 在"环境"下拉框中选择目标环境
2. 环境选项显示完整信息：
   - 环境名称
   - 环境类型标签（dev/test/prod）
   - 集群名称标签
   - 命名空间信息

### 步骤4：填写其他信息
- 版本号（必填）
- 镜像地址（必填）
- 发布策略（滚动/金丝雀/蓝绿）
- 金丝雀比例（如果选择金丝雀发布）
- 发布说明（可选）

### 步骤5：提交发布
- 点击【创建】按钮
- 系统验证所有必填项
- 创建成功后可以提交审批

## 🔍 环境显示格式

环境下拉框中每个选项显示：
```
环境名称 (环境类型) - 命名空间
  [环境类型标签] [集群名称标签]
```

例如：
```
dev-开发环境 (dev) - my-app-dev
  [dev] [本地Kubernetes集群]
```

## ⚠️ 注意事项

### 1. 应用必须先绑定环境
如果应用未绑定任何环境，会提示：
```
该应用还未绑定任何环境，请先在应用详情页绑定环境
```

操作步骤：
1. 进入"应用管理"
2. 选择应用，点击"详情"
3. 在"环境绑定"卡片中点击【绑定环境】
4. 选择环境并配置资源
5. 提交绑定

### 2. 环境字段必选
- 未选择环境时，表单验证会失败
- 错误提示："请选择环境"

### 3. 环境信息自动关联
发布时，系统会使用环境绑定的：
- `namespace`: 部署的命名空间
- `clusterId`: 部署的集群
- `replicas`, `cpuLimit`, `memoryLimit`: 资源配置

## 🔗 与其他功能的集成

### 应用环境绑定
- 位置：应用管理 → 应用详情 → 环境绑定
- 功能：为应用绑定可用环境，配置资源限制
- 关系：发布管理只能选择已绑定的环境

### 环境管理
- 位置：环境与基础设施 → 环境管理
- 功能：创建和管理环境，指定命名空间和集群
- 关系：环境绑定时选择这里创建的环境

### 命名空间隔离
- 设计：每个环境有独立的namespace
- 实现：发布时自动使用环境的namespace
- 好处：不同环境的应用完全隔离

## 📊 数据流向

```
1. 用户选择应用
   ↓
2. 系统查询 app_env_bindings 表
   ↓
3. 加载应用的已绑定环境列表
   ↓
4. 显示环境选项（包含环境、集群、命名空间信息）
   ↓
5. 用户选择环境
   ↓
6. 提交发布（携带 appId + envId）
   ↓
7. 后端根据envId查询环境的namespace和clusterId
   ↓
8. 部署到对应的集群和命名空间
```

## 🎯 API接口

### 查询应用的环境绑定
```
GET /api/v1/app-env-bindings?applicationId={appId}&page=1&pageSize=100
```

响应：
```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "envId": 1,
        "envName": "dev-开发环境",
        "envType": "dev",
        "namespace": "my-app-dev",
        "clusterName": "本地Kubernetes集群",
        "replicas": 2,
        "cpuLimit": "1",
        "memoryLimit": "1Gi"
      }
    ],
    "total": 1
  }
}
```

### 创建发布
```
POST /api/v1/releases

{
  "appId": 1,
  "envId": 1,
  "releaseVersion": "v1.0.0",
  "imageUrl": "nginx:1.26-alpine",
  "releaseStrategy": "rolling",
  "description": "发布说明"
}
```
